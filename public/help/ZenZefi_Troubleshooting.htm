5 ZenZefi Troubleshooting

5.1 ZenZefi cannot be installed or started

ZenZefi cannot be installed or started, or a warning is displayed when ZenZefi during startup.

 

Solution 1: Validate if ZenZefi is running

ZenZefi can be started as an installed Service or as a Java Application.

  • If ZenZefi has been started as a Service (see 2.1.1 ZenZefi installation as a Windows service)

Press <CTRL>+<SHIFT>+<ESC> keys to open the Windows Task Manager and select the tab "Services". Search for "ZenZefi" service and check if is running like in the picture below  

  

  Figure 62: Windows Task Manager - ZenZefi Installed as a Windows Service

  • If ZenZefi has been started as a Java application (see 2.3.1. Start ZenZefi manually via cmd.exe)

1.  Display the command shell window where the Java application has been started.

2.  If the Java application has been started from the console do not close the command shell window. Closing the window will cause ZenZefi to shutdown

Figure 63: ZenZefi Started as a Java Application

  • ZenZefi LogFiles

To get more Information if ZenZefi has stopped running unexpectedly the ZenZefi Logfiles can be evaluated

1.  Navigate to C:\ProgramData\ZenZefi\ServiceLog

2.  Open the file "zenzefi.log"

Solution 3: Delete ZenZefi database

ZenZefi does not start and in the Logfile the following entry can be found:

"Caused by: org.h2.jdbc.JdbcSQLException: Datei fehlerhaft beim Lesen des Datensatzes: "index not found 26". Mögliche Lösung: Recovery Werkzeug verwenden File corrupted while reading record: "index not found 26". Possible solution: use the recovery tool [90030-196]"

 

Solution:

The ZenZefi database is corrupted and must be deleted. Follow the next steps:

1.  Delete all files from C:\ProgramData\ZenZefi\Data

2.  Restart again ZenZefi

5.2 How to start ZenZefi

ZenZefi is installed as a Windows Service, but the service does not start automatically.

Solution 1:  Start ZenZefi manually via cmd.exe

  1. Store zenzefi.jar and zenzefi_cli.exe on your local device.

  2. Open cmd.exe

  

  Figure 65: ZenZefi as a Java Application Start

 

 3.  Navigate to the location where the "zenzefi.jar" and "zenzefi_cli.exe" are stored. E.g. C:\Users\<USERNAME>

 4.  Run the command java -jar zenzefi.jar to start ZenZefi on your local device. (Complete command is C:\Users\USERNAME> java -jar zenzefi.jar)

    • ZenZefi provides a web-based user interface (UI) for R&D. The UI can be accessed by following link: https://localhost:61000/zenzefi/ui/index.html  

    • By default, ZenZefi uses a default user on its startup

Solution 2: Start ZenZefi via double-click

    1.  Store zenzefi.jar and zenzefi_cli.exe on your local device

    2.  Double-click on zenzefi.jar

    3.  To start the ZenZefi UI, open the Browser and point it to URL with following link: https://localhost:61000/zenzefi/ui/index.html

 

i

Single User Mode: After starting, ZenZefi will always run in default user mode. After a user logs in, ZenZefi will run on behalf of that user, no matter over which interface (UI, REST, CLI) the interaction with ZenZefi is triggered.

!

Closing the command shell will stop running ZenZefi, when ZenZefi is started via the CLI!

5.3 How to stop ZenZefi via Windows Task manager

ZenZefi cannot be stopped via CLI, as explained in chapter 2.3 How to stop ZenZefi.

 

Solution:
In some exceptional cases ZenZefi can also be stopped via the Windows Task manager (note: this is not the common way to stop ZenZefi).
When ZenZefi is started from command line, a PID can be noticed in the logs. e.g.: PID 20620

Open Windows Task manager by pressing CRLT + ALT + ENTER and shutdown "javaw.exe *32"or "java.exe" process, which has the same PID as the one from the logs.

  

  Figure 66: Check if ZenZefi "javaw.exe *32" or "java.exe" is Running

 

!

  • By stopping the current running java-process(es), all java sessions will be shut down immediately.

  • The name of the Java process in your task manager may vary depending of the deployed ZenZefi Version.

!

  By stopping ZenZefi via the Windows Task manager, the database can be corrupted. It is recommended to backup the database prior.

5.4 Install the ZenZefi security certificate

When the ZenZefi UI is opened in the browser a warning is displayed, as seen in chapter 1.2.2 Warning messages.

 

Solution:

1.  Open a browser and navigate to ZenZefi UI

2.  In the address bar press "Certificate error"

3.  Press "View certificates"

  

  Figure 67: Untrusted Certificate

 

 4.  In Certificate window press "Install Certificate..."

  

  Figure 68: Install Certificate

 

5.  Press "Next" to start the certificate importing process

  

   Figure 69: Certificate Import Wizard

 

6.  Select "Place all certificates in the following store" and press the "Browse..." button

  

  Figure 70: Certificate Import Wizard Browse Location

 

7.  Select "Trusted Root Certification Authorities" and confirm by pressing the "OK" button

  

  Figure 71: Select Certificate Store

 

8.  Press "Finish" to complete the certificate import process

  

  Figure 72: Complete the Certificate Import Wizard

 

9.  A Security Warning will appear. To confirm the installation of the certificate press "Yes"

  

  Figure 73: Security Warning

 

10.  The import was successful. Press "OK" or close the window by pressing the X button

  

  Figure 74: Import Successful

 

11.  Restart the browser.

5.5 ZenZefi upgrade process fails

ZenZefi can not be upgraded to a newer version.

Solution 1:

Uninstall ZenZefi. See 2.2.1 Uninstall: ZenZefi as a Windows service

 

Solution 2:

Remove the older version from "C:\Program Files\ZenZefi" and copy the new files into this folder again.

  Figure 75: C:\Program Files\ZenZefi

5.6 Error occurred on resolving permissions step

User successfully logged in to ZenZefi (using a Corporate or a Supplier account). While ZenZefi performs an update the message "Error occurred on resolving permissions step" is displayed.

  Figure 76: The User Does Not Have Permissions

This happens because the user does not have user permissions to retrieve user certificates.

Solution:

The user should apply for user permissions in the ACDC.

  • Corporate users (e.g. Daimler internal users): https://cebas-pki.es.corpintra.net/ui/

  • Supplier users: https://cebas-pki.i.daimler.com/ui/

After user permissions have been granted, the error message will not be displayed anymore and the user certificates will be provided.
