1 Introduction

This document provides an overview of the ZenZefi (Zentrale Zertifikatsverwaltung) user functionalities, which are used in Daimler's R&D, Production and After-Sales divisions, as well as, by external suppliers.

1.1 What is ZenZefi?

With the introduction of the Star3 EE-architecture in Mercedes-Benz models, access to Electronic Control Units (ECU) is protected with cryptographic means. Only authorized users, who have the right privileges (encoded in a certificate), are able to access and execute dedicated services on the ECU, e.g. diagnostic- or coding-services.

ZenZefi is an application which is deployed and executed on diagnostic tester-devices in the Daimler business divisions "R&D", "Production" and "After-Sales". In a nutshell, ZenZefi manages user-specific certificates which comprises the creation of Certificate Signing Requests (CSR), validation and selection of certificates, the import and deletion of certificates, the retrieval and update of the certificates, as well as a set of cryptographic functions e.g. chain of trust, creation of signatures or signature validation.

Upon request from client, ZenZefi retrieves certificates and computes signatures that will eventually be propagated to the electronic control unit in the Daimler vehicle. To do so, ZenZefi must access the Public Key Infrastructure (PKI) in the Daimler network.

The cryptographic means used to provide secure access to the vehicle as well as to the control units are based on X.509 (https://en.wikipedia.org/wiki/X.509) certificates that are issued by PKI (https://en.wikipedia.org/wiki/Public_key_infrastructure). Signed by the PKI, the certificate can be used on a diagnostic tester-device as a kind of badge/ID to access services in the ECU. In detail, upon request the certificate is propagated and subsequently verified by the ECU using the Root- and Backend-Certificates (which are stored inside the ECU). According to the role of the user, which can be determined from the certificate, the control unit will allow access to the diagnostic service provided by the control unit.

1.5 System requirements

For Daimler-internal use, ZenZefi can be deployed on Windows and Linux-based testing devices. Depending on the business area, these testing devices operate with different vehicle diagnostic applications such as Monaco in R&D, IS-Service+ in production and Xentry Diagnostics in After-Sales.

Due to the different user communities, runtime environments for ZenZefi may range from full-fledged computers to various testing devices from different vendors. Therefore, hardware requirements may vary.

i

Mandatory system requirement for production: ZenZefi requires at least a Java 8 32bit or 64bit version which needs to be installed.

Java is available on https://www.java.com, https://adoptopenjdk.net, https://aws.amazon.com/corretto.

The installers for Enduser and After-Sales have a Java runtime included and don’t need an additional Java installation.

Please mind that If ZenZefi or the ZenZefi command line tools (also the .exe cli tools) are used from command line a Java executable needs to be available on the system path.

1.6 Minimum system requirements

1.6.1 Hardware

  • 100 MB free disk space for program installation

  • 300 MB for log files

  • 200 MB for Database files

  • 400 MB free RAM for execution

1.6.2 Operating systems

  • Windows 7 or Windows 10 (32 bit or 64 bit)

  • Debian Linux 8 (“Jessie”), 64 bit AMD64

  • Write access to the user home directory (to create the database)

  • Write access to the ZenZefi installation directory (to write log-files, directory can be configured)

  • Free local port 61000

  • Outgoing port 443 to connect to Daimler Backend for Certificate download

1.6.3 Installed software

ZenZefi requires the following software to be installed (32 bit or 64 bit)

  • .NET4.5 is required to run the MSI installer on Windows. In general ZenZefi does not require .NET4.5

  • Java has to installed for production installations:

    • Oracle Java Cryptography Extension (JCE) Unlimited Strength (https://www.oracle.com/technetwork/java/javase/downloads/jce8-download-2133166.html)

    • Oracle Java 1.8 (https://www.oracle.com/technetwork/java/javase/downloads/index.html) or alternatively

      • Open JDK 8 (https://openjdk.java.net/)

      • Amazon Corretto 8 (https://aws.amazon.com/corretto)

      • any other java version that is fully compatible to Oracle Java 1.8

  • Java is also needed for all command line tools or if ZenZefi is started from command line

1.6.4 Valid time of the operating system

ZenZefi needs a valid system time of the operational system, which is used to automatically update and delete certificates within the user store. A wrong system time could cause problems for the automated certificate management of ZenZefi.

1.6.5 Browser to display the ZenZefi UI

A modern browser, like Chrome or Firefox, is recommended to work with the ZenZefi UI. Moreover, ZenZefi also supports Microsoft Edge.

1.6.6 Firewall settings

Access to port 443 must be allowed as the ZenZefi uses HTTPS to communicate with the PKI to retrieve certificates ZenZefi communicates with Daimler's PKI. Therefore, port 443 must be activated for outgoing connections inside firewall rule sets.

The following URLs must also be activated for outgoing connections inside the firewall rule set:

- After-Sales: https://cebas-pki.i.daimler.com

- Supplier: https://cebas-pki.i.daimler.com

- Emloyee: https://cebas-pki.emea.svc.corpintra.net

- For all variants: https://sso.daimler.com
