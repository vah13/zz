4 ZenZefi commands for the Command Line Interface (CLI)

The following table contains all ZenZefi commands that can be executed via the CLI:

 

i

Further commands might be introduced in the next ZenZefi versions. Please check the available commands via command line parameter -help.

 

Please execute following command via CLI: (Navigate to the directory where the "zenzefi_cli.exe" is located: C:\Program Files\ZenZefi) zenzefi_cli.exe <COMMAND> (from table below)

 

Command

Description

Example

-all

Optional command that specifies deleting the entire certificates store. It is use only with command: '-deleteCertificatesAkiSn'

 

-cert <authorityKeyIdentifier,serialNumber>

Specifies a pair of authorityKeyIdentifier and serial number. It is used only with command: '-deleteCertificatesAkiSn'

<authorityKeyIdentifier> = authority key identifier of the regarding certificate

<serialNumber> = serial number of the regarding certificate

-cert 0E-A1-92-D1-DD-26-9B-B8-79-80-A9-1A-A2-A7-61-C9-0E-A5-34-FB,01

-checkActiveDiagCert <backendCertSkid> <diagCertSerialNo>

Optional parameters: <vin>, <ecu>, <userRole>

Checks Active Diagnostic Certificate based on given criteria (vin, ecu and userRole are optional).

<backendCertSkid> = subject key identifier of backend certificate

<diagCertSerialNo> = serial number of the regarding diagnostics certificate

<vin> = vin of the regarding diagnostics certificat

<ecu> = ecu the regarding diagnostics certificate

<userRole> = user role of the regarding diagnostics certificate

-checkActiveDiagCert 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 08-00-00-01-vin v1 -ecu e1 -userRole 1

-checkOwnership <challenge> <backendCertSkid> <serialNo>

Optional parameters: <outputdir>

Checks ownership based on given criteria. ZenZefi selects internally the diagnostics certificate (PKIRole 03) where the serial number of the certificate matches the SerialNumber from the call and where the authority key identifier matches the BackendCert-Subject-KeyIdentifier from the call. If no certificate or more than one certificate are found, an error will be returned.

If one certificate is found, ZenZefi selects the private key for this certificate in its internal storage and calculates a signature for the ECUChallenge using this private key.

Custom directory can be specified, if not it will be saved in the current directory.

<challenge> = path to ECUChallenge file

<backendCertSkid> = subject key identifier of backend certificate

<serialNo> = serial number of the regarding certificate

<outputdir> = custom directory

-checkOwnership C:/folder/data.txt 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 08-00-00-01 -outputdir C:/folder

-checkSystemIntegrity

Makes system integrity check. Custom path can be specified.

-checkSystemIntegrity -path C:/folder

-checkVersion

Checks REST and project version.

 

-deleteCertificates <ids>

Deletes certificates by ids. The ids list can contain spaces, provided that the list is between quotation marks.

<ids> = id(s) of the regarding certificate(s)

-deleteCertificates3b1cc183-af06-4b64-9e0e-aa837219f754,b1cc183-af06-4b64-9e0e-aa83721aa454

-deleteCertificatesAkiSn

Optional parameters: <cert>

Delete all certificates by using option -all or delete certificates by authority key identifier and serial number. The authority key identifier/ serial number pair can contain spaces, provided that the pair is between quotation marks.

If a root certificate needs to be deleted, the syntax is: '-cert ,sn', due to the fact that root certificates do not have authority key identifier, and the syntax for the command line must be consistent.

<cert> = is composed from authority key identifier of the regarding certificate and serial number of the regarding certificate. They are separated by comma.

-deleteCertificatesAkiSn -cert aki1,sn1 -cert aki2,sn2 -cert aki3,sn3

-differentialUpdate

Performs differential update of the certificates. Only those certificates in the user certificate store are requested which will expire shortly or which are not available yet in user certificate store.

 

-ecu <ecu>

Optional command that specifies the targetECU. It is used only with commands: '-getDiagCert, -checkActiveDiagCert,-getEnhDiagCert, -getTimeCert, -secureVariantCoding'

-ecu ECUDiagnoseName

-enableAutologout <state>

Enable/Disabled automatically logout

  <state> = “ON” or “OFF”

-enableAutologout ON

-fullUpdate

Performs full update of the certificates. All certificates the user has rights for are requested independently of the certificate renewal time.

 

-getCertByID <authorityKeyIdentifier> <serialNo>

Optional parameters:<outputdir>

Gets and saves into file certificate by authority key identifier and serial number. Custom directory can be specified, if not it will be saved in the current directory.

<authorityKeyIdentifier> = authority key identifier of the regarding certificate

<serialNo> = serial number of the regarding certificate

<outputdir> = custom directory

-getCertByID 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 08-00-00-01 -outputdir C:/folder

-getDiagCert <backendCertSkid>

Optional parameters: <vin>, <ecu>, <userRole>, <outputdir>

Gets and saves into file certificate based on given criteria (vin, ecu and userRole are optional). Custom directory can be specified, if not it will be saved in the current directory.

<backendCertSkid> = subject key identifier of backend certificate

<vin> = vin of the regarding diagnostics certificate

<ecu> = ecu of the regarding diagnostics certificate

<userRole> = user role of the regarding diagnostics certificate

-getDiagCert 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 -vin v1 -ecu e1 -userRole 1 -outputdir C:/folder

-getDiagCertForCentralAuthentication <backendCertSkid>

Optional parameters: <vin>, <outputdir>

Gets and saves into file certificate based on given criteria (vin, is optional). Custom directory can be specified, if not it will be saved in the current directory.

<backendCertSkid> = subject key identifier of backend certificate

<vin> = vin of the regarding diagnostics certificate

<outputdir> = custom directory

-getDiagCertForCentralAuthentication 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 -vin WDD2231234A123456 -outputdir C:/folder

-getDiagCertIdRole <backendCertSkid> <serialNo>

Gets the role of a diagnostic certificate searched by subject key identifier and serial number.

<backendCertSkid> = subject key identifier of backend certificate

<serialNo> = serial number of the regarding certificate

-getDiagCertIdRole 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 08-00-00-01

-getEnhDiagCert <backendCertSkid> <diagCertSerialNumber>

Optional parameters: <vin>, <ecu>, <outputdir>

Gets and saves certificates into a file based on given criteria (vin and ecu are optional). Custom directory can be specified, if not it will be saved in the current directory.

<backendCertSkid> = subject key identifier of backend certificate

<diagCertSerialNumber> = serial number of the regarding diagnostics certificate

<vin> = vin of the regarding enhanced certificate of the regarding enhanced certificate

<outputdir> = custom directory

-getEnhDiagCert 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 08-00-00-01 -vin v1 -ecu e1 -outputdir C:/folder

-getEnhRightID <services> <backendSKI>

Optional parameters: <outputdir>

Determine Enhanced Rights Certificates for dedicated services. Search for Enhanced Rights certificates in the user store containing specific services. Custom directory can be specified, if not it will be saved in the current directory.

<outputdir> = custom directory

-getEnhRightID 1234,ffee,ff D3-2E-9A-4B-64-B1-9A-8A-DE-74-7F-31-D2-77-F9-81-94-00-AE-DE -path C:/folder

-getLogArchive <path>

Downloads zenzefi-logs.zip to the current directory or to the specified path.

-getLogArchive -path C:/folder

-getLogFile <path>

Downloads zenzefi-database-logs.xml to the current directory or to the path specified.

-getLogFile -path C:/folder

-getReplacementPackage <ecuCertificatePath> <backendSKI>

Optional parameters: <uniqueEcuId>, <outputdir>

Retrieves a certificate replacement package (unique ecu id is optional). Accepted target values: ROOT, BACKEND, ECU.

<ecuCertificatePath> = path for ECU certificate
<backendSKI> = subject key identifier of backend certificate

<uniqueEcuId> = Unique ECU ID

<outputdir> = custom directory

-getReplacementPackage C:/folder/ecu.crt D3-2E-9A-4B-64-B1-9A-8A-DE-74-7F-31-D2-77-F9-81-94-00-AE-DE -uniqueEcuIdABR123 -outputdir C:/folder

-getRootBackends <target>

Request all Backend CA certifictes (BACKEND) or all Root CA certificates (ROOT). If no parameter is given all Backend CA certificates and all Root CA certificates are provided. Certificates are printed in base64 format to STDOUT (console).

-getRootBackends ROOT

-getSecOCISCert <certificate> <backendCertSkid> <diagCertSerialNo> <ecu> <vin>

Optional parameters: <outputdir>

Gets and saves into file certificate based on given criteria (vin is mandatory). The certificate will be read from the provided file. Custom directory can be specified, if not it will be saved in the current directory.

<certificate> = path of the regarding certificate

<backendCertSkid> = subject key identifier  of backend certificate

<diagCertSerialNo> = serial number of the regarding diagnostics certificate

<ecu> = ecu of the regarding SecOCIS certificate

<vin> = vin of the regarding SecOCIS certificate

<outputdir> = custom directory

-getSecOCISCert C:/folder/ABR223.crt 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 01-00-00-01 EZS223 -vin v1 -outputdir C:/folder

-getTimeCert <backendCertSkid> <nonce> <vin>

Optional parameters: <outputdir>

Gets and saves into file certificate based on given criteria (ecu is optional). Custom directory can be specified, if not it will be saved in the current directory.

<backendCertSkid> = subject key identifierof backend certificate

<nonce> = nonce of the regarding time certificate

<vin> = vin of the regarding time certificate

<outputdir> = custom directory

-getTimeCert 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 12-34-56-78-90-AB-CD-EF-12-34-56-78-90-AB-CD-EF-12-34-56-78-90-AB-CD-EF-12-34-56-78-90-AB-CD-EF WDD2231234A123456 -ecu e1 -outputdir C:/folder

-help

Contains the description and samples for each command

 

-importCert <filePath,password>

Imports the certificate or PKCS12 container file passed as argument. If a password is needed, it will be given separated by a comma. If an empty password is needed, the file path will end with a comma.

-importCert D:/SVN/ZenZefi/zenzefi-cli/target/severalCerts.p12,secret

-importProductionCert

 

Imports the encrypted PKCS Package and backend subject key identifier passed as argument. The parameters are comma separated.

-importProductionCert  -package BE0zIrC....wxTWNQ=

-listCertificates

Optional parameter: <outputdir>

Saves into file the list of certificates for the current logged in user. Custom directory can be specified, if not it will be saved in the current directory.

<outputdir> = custom directory

-listCertificates -outputdir C:/folder

-listPerformanceAuditEntries

Saves into file the list of performance audit entries. Saves in the current directory.

 

-logout

Logs out user.

 

-outputdir <arg>

Specifies the directory where the data file will be saved. It is used with '-getCertByID, -getEnhDiagCert, -getTimeCert, -secureVariantCoding, -getSecOCISCert'

-outputdir C:/ProgramData/ZenZefi/output

-package <package>

Specifies a pair of encrypted Base64 pkcs package and encrypted backend subject key identifier.

-package BH5oU5cslU0zMltS8y85EOy4cY....

-partNumberMapping <BSKI/PartNumber>

 

-partNumberMapping  1234567891

-path <arg>

Specifies the path where it will save the result of the command.

-path C:/ProgramData/ZenZefi/output

-proxyType <proxyType>

Specifies the proxy type.

-proxyType DIRECT

-randomNonce

Commands for generating secure random nonce for tester.

 

-restoreCert

Restores root and backend certificates (if these have been deleted from the user certificate store) that were deleted from user certificate store.

 

-secureVariantCoding <backendCertSkid> <data>

Optional parameters: <vin>, <ecu>, <outputdir>

Saves into file certificate and signature based on given criteria, by retrieving the private key that belongs to this certificate and signing the provided data file (vin and ecu are optional). Custom directory can be specified, if not it will be saved in the current directory.

<backendCertSkid> = subject key identifier  of backend certificate

<data> = path to data

<vin> = vin of the regarding certificate

<ecu> = ecu of the regarding certificate

<outputdir> = custom directory

-secureVariantCoding 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 C:/folder/data.txt -vin v1 -ecu e1 -outputdir C:/folder

-serialNo <serialNo>

Specifies the serial number.

-serialNo 02-00-00-01

-serverPort <arg>

Specifies the port on which a command can be run.

-serverPort 61000 -logout

-setLogLevel  <logLevel>

Sets the log level for loggers. Only logs with configured log level and the levels above will be displayed. Available levels: INFO, FINE, FINER, FINEST

-setLogLevel FINE

-setProxy <arg>

Sets the proxy to be used in ZenZefi. The proxy type can be DIRECT, MANUAL, NATIVE.

<arg> = proxy. It is formed by the hostname and the port.

-setProxy service.com 8080 -proxyType MANUAL

-shutdown

Shuts down the application.

 

-signatureCheck <message> <signature> <ecuCertificate>

Checks signature for the provided message parameter and ECU Certificate.

<message> = path to message file

<signature> = path to signature file

<ecuCertificate> = path to ecu certificate

-signatureCheck -signatureCheck signatureCheck/data.txt C:/folder/newSig.sig C:/folder/BC_F223.crt

 -signaturesAllBackends <data>

Optional parameters: <vin>, <ecu>, <outputdir>

Signs coding data with variant coding user certificates found under all available backends (vin and ecu are optional).

<data> = data

<vin> = vin of the regarding certificate

<ecu> = ecu of the regarding certificate

<outputdir> = custom directory

-signaturesAllBackends C:/folder/data.txt -vin WDD2231234A123456 -ecu ECU123 -outputdir C:/folder

 -signECU <challenge> <backendCertSkid> <ecuID>

Optional parameters: <outputdir>

Signs a challenge byte-array with the private key of an ECU certificate. The serial number and outputdir are optional.

<challenge> = path to ECUChallenge file

<backendCertSkid> =subject key identifier  of backend certificate

<ecuID> = ECU ID

<outputdir> = custom directory

-signECU C:/folder/data.txt 51-82-B1-C2-D6-8C-F7-9A-52-EC-83-4A-1A-80-82-8F-0C-53-58-D4 ABRL229 -serialNo 02-00-00-01 -outputdir C:/folder

-systemsPublicKey

Gets systems public key.

 

-uniqueEcuId <uniqueEcuId>

Specifies the unique ECU ID. It is used with the -getReplacementPackage command.

-uniqueEcuId ABR123

-userRole <arg>

Optional command that specifies the user role numeric value. It is used only with commands: '-getDiagCert,-checkActiveDiagCert'. Values: 1 Supplier, 2 Development ENHANCED, 3 Production, 4 After-Sales ENHANCED, 5 After-SalesSTANDARD, 6 After-Sales BASIC, 7 Internal Diagnostic Test Tool, 8 ePTI Test Tool. The order of the values can beconfigured from user settings. This is the default order.

-userRole 1

-validateCertificates <state>

Enable/Disabled extended validation

  <state> = “ON” or “OFF”

-validateCertificates ON

-version

Get ZenZefi version.

 

-vin <vin>

Optional command that specifies the targetVIN. It is used only with commands: '-createCertSignReq',-checkActiveDiagCert, -getDiagCert, -getEnhDiagCert, -secureVariantCoding

-vin WDD2231234A123456
